# 銀行レート比較ページ：みずほレートの更新手順（ルーティン用）

このディレクトリの `rates.json` は、GitHub Actions（`.github/workflows/fetch-rates.yml`）が
平日10:15/12:05/15:05ごろに自動更新している。ただし**みずほ銀行はデータセンターIPからの
アクセスを遮断しているため、Actionsでは取得できない**。みずほの値だけは、この手順に
従って更新する（Claudeルーティンまたは手動）。

## 更新のルール

1. 情報源：みずほ銀行の公表する対顧客外国為替相場（TTS/TTB）。
   https://www.mizuhobank.co.jp/market/index.html （またはそこからたどれる相場一覧）。
   **公式ページで実際に確認できた値のみ**を書くこと。確認できなければ既存値を変更しない
   （古い値は fetched の日時から古さが分かるので、そのままで良い）。
2. 対象通貨：USD / EUR / GBP / CNY / TWD / KRW / VND のうち、みずほが公表しているもの。
3. **値はすべて「1通貨あたりの円」で記録する**。みずほの公表表でウォン・ドンが
   「100通貨あたり」で示されている場合は **100で割ってから** 書く。
   - 妥当レンジの目安（これを外れたら単位間違いを疑う）:
     USD 60〜400 / EUR 80〜400 / GBP 100〜500 / CNY 10〜60 / TWD 2〜15 /
     KRW 0.03〜0.4 / VND 0.002〜0.02
   - TTS > TTB になっているはず（逆なら取り違え）。
4. `rates.json` の `banks.mizuho` **だけ**を編集する。他の銀行・`updated`・`currencies`
   には触れない。形式:
   ```json
   "mizuho": {
    "rates": {
     "USD": {"tts": 163.05, "ttb": 161.05},
     "EUR": {"tts": 186.50, "ttb": 183.50}
    },
    "fetched": "YYYY-MM-DD HH:MM"
   }
   ```
   `fetched` は取得時刻（日本時間）。
5. 編集後は main ブランチにコミット＆プッシュする（GitHub Pagesが自動で再デプロイする）。
   コミットメッセージ例: `Update Mizuho FX rates (manual/routine)`

## 注意

- Actions側のスクリプト（`scripts/fetch_rates.py`）は「取得に失敗したソースの前回値を
  維持する」ため、ここで書いたみずほの値はActionsの自動更新後も消えない。
- レートが確認できない日（祝日・障害など）は何もしない。誤った値を書くより
  古い値のままの方が安全。
